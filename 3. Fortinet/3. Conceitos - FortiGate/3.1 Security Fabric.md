```bash
# Verificar membro upstream (root)

```

### 1.4 Configuração


```bash
# Desativar sincronização de configuração upstream → downstream
config system csf
    set configuration-sync local
end

# Desativar transferência de objetos do fabric
config system csf
    set fabric-object-unification local
end
```

## 1) Conceito

O **Security Fabric** é a arquitetura de integração entre dispositivos e soluções Fortinet, permitindo:

- **Visibilidade**
- **Integração**
- **Automatização**

---

## 2) Estrutura da malha

### 2.1 Topologia base

- **Root** → nó principal do fabric
- **Upstream** → direção principal da hierarquia
- **Downstream** → nós subordinados que recebem sincronização

### 2.2 Exemplo de núcleo do fabric

- **2 FortiGate + FortiAnalyzer** → cenário **Core**

---

## 3) Classificação dos componentes

### 3.1 Core

Exemplo de núcleo da arquitetura:

- **2 FortiGate + FortiAnalyzer**

### 3.2 Recommended

Componentes fortemente recomendados no fabric:

- **FortiManager**
- **FortiAP**
- **FortiSwitch**
- **FortiClient**
- **FortiSandbox**
- **FortiMail**

### 3.3 Extended

Integrações ampliadas:

- Outros produtos Fortinet
- Produtos de terceiros usando **API**

---

## 4) Direção de sincronização

### Termos principais

- **Root** → origem da sincronização
- **Upstream** → camada superior da malha
- **Downstream** → dispositivos que recebem a configuração

### Regra principal

- **Toda configuração de objeto é replicada do root para os downstream**

Isso inclui, por exemplo:

- objetos de endereço
- grupos de endereço
- serviços
- regras/elementos compartilhados pelo fabric

---

## 5) Resumo curto para revisão

- O **Security Fabric** organiza a rede Fortinet em uma malha integrada
- Tem foco em **visibilidade, integração e automação**
- **Root** é o ponto central da replicação
- **Downstream** recebe as configurações do root
- Componentes podem ser classificados como:
    - **Core**
    - **Recommended**
    - **Extended**
