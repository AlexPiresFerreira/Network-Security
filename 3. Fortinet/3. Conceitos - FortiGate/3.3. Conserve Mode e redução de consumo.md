
### 4.1 Visão geral

- O **Conserve Mode** é acionado quando a memória fica crítica.
- A ideia é reduzir consumo, aliviar inspeção e evitar colapso do sistema.

### 4.2 Redução do tamanho de inspeção


```bash
config firewall profile-protocol-options
    edit <profile_name>
        config [http | ftp | pop3 | smtp | imap]
            set oversize-limit <MB>
        end
    next
end
```

- Reduz o limite de arquivo para inspeção
- Útil quando o ambiente trabalha com arquivos menores do que o padrão

### 4.3 Redução de cache

#### FortiGuard

```bash
config system fortiguard
    set webfilter-cache-ttl <sec>
    set antispam-cache-ttl <sec>
end
```

#### DNS


```bash
config system dns
    set dns-cache-ttl <sec>
end
```

### 4.4 Redução de TTL de sessão

#### Global

```bash
config system session-ttl
    set default <sec>
end
```

#### UDP global

```bash
config system global
    set udp-idle-timer <sec>
end
```

#### Por serviço

```bash
config system session-ttl
    config port
        edit <id>
            set protocol <IP_protocol>
            set start-port <start_port>
            set end-port <end_port>
            set timeout <sec>
        next
    end
end
```

#### Por policy

```bash
config firewall policy
    edit <id>
        set session-ttl <sec>
    next
end
```

### 4.5 Timers TCP globais

```bash
config system global
    set tcp-halfclose-timer <sec>
    set tcp-halfopen-timer <sec>
    set tcp-timewait-timer <sec>
end
```

- **tcp-halfclose-timer**: tempo para sessão sem FIN/ACK após FIN
- **tcp-halfopen-timer**: tempo para sessão sem SYN/ACK após SYN
- **tcp-timewait-timer**: tempo de permanência após fechamento da conexão

### 4.1 Conceito

- O FortiGate entra em **Conserve Mode** quando a memória atinge níveis críticos
- Inspeções pesadas são reduzidas automaticamente para proteger o sistema
