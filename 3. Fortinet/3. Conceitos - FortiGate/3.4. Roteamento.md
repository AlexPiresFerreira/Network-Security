## 7) Rota default

- Rota default: **destino 0.0.0.0/0.0.0.0**
- Usada para tráfego:
    - não diretamente conectado
    - não conhecido por rotas específicas

### 7.1 Configuração

- Informar:
    - **gateway** (ou **gateway dinâmico** se interface é DHCP/PPPoE)
    - **interface de saída**
    - distância administrativa
    - prioridade

### 7.2 Distância x prioridade

- Quando há **mais de uma rota** para o mesmo destino:
    - **menor distância** → preferência
    - se distância igual: **menor prioridade** → preferência

## 1) O que é roteamento IP no FortiGate

- No modo **NAT**, o FortiGate atua como **roteador IP L3**.
- Ele encaminha pacotes entre redes IPv4 e IPv6.
- O roteamento acontece:
    - no tráfego passando pelo firewall
    - no tráfego de saída local do próprio FortiGate
- A decisão de roteamento define:
    - **interface de saída**
    - **gateway / next hop**

---

## 2) Como o FortiGate escolhe a rota

- A seleção da melhor rota considera:
    - **rota mais específica**
    - **distância administrativa**
    - **métrica**
    - **prioridade**
- Se houver múltiplas rotas para o mesmo destino:
    - o FortiGate usa os atributos da rota para desempate
- O roteamento vem **antes** da maioria das ações de segurança:
    - por isso, a política deve ser pensada com base no roteamento

---

## 4) Rotas estáticas

- São configuradas manualmente.
- Servem para:
    - rota default
    - redes específicas
    - interligação entre sub-redes

### Exemplo de uso

- `0.0.0.0/0` como rota default
- objetos de endereço do tipo:
    - **Subnet**
    - **FQDN**
- também podem ser usados como destino em rotas estáticas

---

## 5) Tabela de roteamento

- A tabela mostra apenas:
    - rotas conectadas
    - rotas estáticas
    - rotas dinâmicas ativas
- Não mostra:
    - rotas inativas
    - rotas em espera
    - policy routes

### Visualização de policy routes

- Ao trocar a visualização para **Policy**, aparecem:
    - policy routes
    - rotas ISDB
    - regras de SD-WAN

---

## 6) Distância, métrica e prioridade

### Distância

- É o primeiro critério de desempate entre rotas duplicadas.
- **Menor distância = maior preferência**
- Rotas com maior distância podem ficar em espera.

### Métrica

- É usada como critério de desempate entre rotas dinâmicas do mesmo protocolo.
- **Menor métrica = maior preferência**

### Prioridade

- É o desempate para rotas estáticas em ECMP.
- **Menor prioridade = maior preferência**
- Valor padrão: **1**

---

## 7) Ferramenta de procura de rota

- O FortiGate permite procurar rota por:
    - destino
    - porta de destino
    - origem
    - porta de origem
    - protocolo
    - interface de origem
- Se todos os critérios forem informados:
    - ele consulta também a tabela de **Policy Route**
- Caso contrário:
    - consulta só a tabela de roteamento

---

## 8) Reverse Path Forwarding (RPF)

- É uma proteção contra **IP spoofing**
- O IP de origem é validado para verificar se existe caminho de retorno
- A checagem é feita apenas no **primeiro pacote da sessão**

### Modos

- **Feasible path**
    - padrão
    - o caminho de retorno não precisa ser a melhor rota
- **Strict**
    - o caminho de retorno precisa ser a melhor rota

### Quando falha

- O debug mostra algo como:
    - `reverse path check fail, drop`

## 9) ECMP

- ECMP significa **Equal Cost Multi Path**
- Ocorre quando existem múltiplas rotas com mesmo custo.
- As rotas ECMP são instaladas na **RIB**
- O tráfego é balanceado entre os caminhos

### Algoritmos de balanceamento

- **Source IP**
    - padrão
    - sessões do mesmo IP usam a mesma rota
- **Source-Destination IP**
    - mesma origem + mesmo destino usam a mesma rota
- **Weighted**
    - apenas para rotas estáticas
    - usa peso da rota ou da interface
- **Usage / Spillover**
    - usa uma rota até atingir o limite de banda e depois passa para a próxima

## 10) SD-WAN x ECMP

- Quando o **SD-WAN** está habilitado:
    - o FortiOS esconde o `v4-ecmp-mode`
    - e passa a usar `load-balance-mode` no `config system sdwan`
- O SD-WAN:
    - usa pesos dos membros
    - usa thresholds dos membros
    - suporta algoritmo de volume
- O ECMP tradicional:
    - usa peso da rota estática ou da interface
    - não suporta volume algorithm

- **RIB (Routing Information Base)**
    
    - Tabela **completa** de informações de roteamento:
    - rotas estáticas
    - rotas dinâmicas (OSPF, BGP, RIP etc.)
    - rotas aprendidas via SD‑WAN / policy route
    - Usada como “banco de dados” de todas as rotas conhecidas.
- **FIB (Forwarding Information Base)**
    
    - Versão **otimizada** da RIB, usada pelo FortiGate para:
    - tomada de **decisões de encaminhamento em tempo real**
    - lookup rápido (FIB / kernel)
    - Contém apenas as rotas efetivamente **instaladas** para forwarding.
- **Em resumo:**
    
    - **RIB** = visão completa de roteamento.
    - **FIB** = subconjunto otimizado, usado no caminho de dados para encaminhar pacotes.

- O arquivo inclui tópicos de **OSPF** e **BGP**, sugerindo uma revisão dos protocolos dinâmicos de roteamento.
- A presença desses tópicos indica foco em:
    - troca de rotas
    - decisão de caminho
    - integração com cenários corporativos e VPN

**Ideia central:** relacionar roteamento dinâmico com o funcionamento do firewall e com a escolha do melhor caminho de tráfego.
