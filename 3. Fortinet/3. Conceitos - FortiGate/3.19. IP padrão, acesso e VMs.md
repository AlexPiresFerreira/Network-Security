### 1.2 Usuário e senha padrão

- **Usuário**: `admin`
- **Senha**: em branco (sem senha)
- Na **primeira autenticação** é obrigatório trocar a senha  
    → sempre usar senha forte.

## 3) Acesso via console e CLI

- Acesso **sem rede** via porta **console**
    - Cabo RJ45–serial ou RJ45–USB
    - Programas: **PuTTY**, **Tera Term**, etc.
- Métodos de administração:
    - **GUI** (HTTP/HTTPS)
    - **CLI** (console/SSH/Telnet, widget “CLI Console”)
- Observações:
    - Relatórios só via **GUI**
    - Muitos comandos avançados e diagnósticos só na **CLI**
    - SNMP é **read-only**

## 9) Métodos de administração e perfis

### 9.1 Métodos

- **GUI**:
    - navegador (HTTP/HTTPS)
- **CLI**:
    - widget CLI Console
    - SSH/Telnet
    - porta console

### 9.2 Criação de administradores

- Recomenda-se:
    - **um usuário por administrador**
    - maior segurança e rastreabilidade
- Suporte a:
    - usuários locais
    - usuários remotos via **LDAP, RADIUS, TACACS+**
    - autenticação por **certificado digital**

### 9.3 Perfis de admin

- **super_admin**:
    - acesso total (global)
    - usado pela conta `admin`
    - não pode ser alterado
- **prof_admin**:
    - acesso completo **por VDOM**
    - não pode alterar config global
- Perfis customizados:
    - permissões **Read/Write**, **Read-Only** ou **None** por área
    - podem ter timeout de inatividade customizado

### 9.4 Trusted Hosts

- Por usuário admin:
    - lista de IPs/redes permitidos para acesso administrativo
- Exemplo:
    - `192.168.99.0/24` → só essa rede consegue logar com aquele admin
    - `0.0.0.0/0` → qualquer origem (menos seguro)

### 9.5 Configurações administrativas gerais

- Portas de administração customizáveis
- Recomendado:
    - **HTTPS e SSH** apenas
- Timeout de inatividade:
    - default **5 minutos**
    - pode ser ajustado globalmente ou por perfil admin
- Protocolos por interface:
    - gerenciamento IPv4 e IPv6
    - Security Fabric connection (CAPWAP, FortiTelemetry, FMG-Access, FTM, RADIUS Accounting, LLDP)
