
### Debug flow (rastreamento de tráfego)

```bash
# 1. Definir filtro (usar addr, saddr, daddr, sport ou dport)
diagnose debug flow filter addr <IP>

# 2. Iniciar rastreamento (definir número de pacotes)
diagnose debug flow trace start <número_de_pacotes>

# 3. Habilitar debug
diagnose debug enable
```


```
# Habilitar exibição de nomes de funções
diagnose debug flow show function-name enable

# Aplicar filtro
diagnose debug flow filter <filtro>

# Habilitar debug
diagnose debug enable

# Iniciar rastreamento
diagnose debug flow trace start <count>

# Parar rastreamento
diagnose debug flow trace stop
```

### 13.2 Uso

- Segue:
    - criação de sessão
    - consulta de rota
    - aplicação de policy
    - NAT
    - forward
- Útil para:
    - pacote sem passagem
    - NAT errado
    - rota incorreta
    - policy não casando
### Sequência recomendada

1. Aplicar filtro
2. Habilitar debug
3. Iniciar trace
4. Reproduzir o problema
5. Parar trace
6. Desabilitar debug

- Útil para mapear o caminho do pacote
- Ajuda a identificar:
    - policy match
    - roteamento
    - NAT
    - drops internos

###  Sniffer de pacotes

```bash
diagnose sniffer packet <interface> '<filter>' <verbose> <a(UTC)|l(local)>
```

- `a` → Absolute UTC time
- `l` → Local time
- Níveis de verbose

| Nível | O que mostra         |
| ----- | -------------------- |
| `1`   | Cabeçalho básico     |
| `2`   | Cabeçalho IP         |
| `3`   | Cabeçalho IP + dados |
| `4`   | Cabeçalho detalhado  |
| `5`   | Detalhado + hex      |
| `6`   | Máximo               |


> **Verbose recomendado:** `4` (inclui cabeçalho + payload)
> - `in` → tráfego **entrando** na interface
> - `out` → tráfego **saindo** da interface
