### ⚠️ **Dica Importante: Conserve Mode**

- **O que é:** Um estado de alerta crítico onde o FortiGate começa a liberar recursos agressivamente para evitar uma falha total, impactando a performance e a conectividade. Ele tenta preservar a memória RAM para funções críticas, descartando pacotes e desabilitando algumas funcionalidades. É um sinal de que o FortiGate está sob extrema pressão de memória.

- **Comando para verificar:** `diagnose hardware sysinfo conserve`
- **Saída Exemplo:**

```
  memory usage: 20%
  red zone: 80%
  green zone: 70%
  ...
  in conserve mode: no
```

- `memory usage`: Uso atual da memória.
    
- `red zone`: Limite percentual de memória onde o FortiGate entra em "conserve mode" crítico (descarte de pacotes, desabilitação de AV/IPS).
    
- `green zone`: Limite percentual de memória abaixo do qual o FortiGate sai do "conserve mode".
    
- `in conserve mode`: Indica se o FortiGate está atualmente em conserve mode (`yes` ou `no`).
    
    - **Configurar Thresholds:** Você pode ajustar os limites de memória para o conserve mode, mas isso deve ser feito com cautela e entendimento dos riscos.

forticli

```forticli
  config system global
      set memory-use-threshold-red 80  # % para entrar em modo vermelho (alerta crítico)
      set memory-use-threshold-green 70 # % para sair do modo verde (normal)
      set av-failopen-session enable # Permitir sessões AV passarem se em conserve mode
  end
```

- `av-failopen-session enable`: Em conserve mode, o FortiGate pode parar de inspecionar tráfego AV para economizar memória. Esta opção permite que o tráfego passe sem inspeção AV, em vez de ser bloqueado.
    
    - **Boas Práticas:**
    - Monitore o "Conserve Mode" ativamente. Um FortiGate entrando em `red-mode` frequentemente é um sinal de que está subdimensionado ou mal configurado.
    - Se o FortiGate entrar frequentemente em Conserve Mode, investigue a causa (excesso de sessões, bugs de firmware, recursos insuficientes, ataques DoS).
    - Considere aumentar os recursos de hardware (CPU/RAM) ou otimizar as políticas/perfis de segurança (ex: desabilitar features não utilizadas, otimizar inspeção SSL, reduzir o número de VDOMs).
    - Em casos extremos, se um processo específico estiver causando o conserve mode, você pode tentar matá-lo com `diagnose sys kill 9 <PID>`, mas isso pode causar instabilidade. Um reboot pode ser necessário.
