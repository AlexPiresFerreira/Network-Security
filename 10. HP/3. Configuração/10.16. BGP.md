### Distancia AD
 
Para configurar a distancia administrativa dentro processo BGP ou dentro do processo “ipv4-family 
vpn-instance [nome da vrf]” no BGP use a sintaxe:

```
[Router-bgp]preference ?
INTEGER<1-255>  External preference
!Distancia administrativa para rotas aprendidas via eBGP
 
[Router-bgp]preference 7 ?
INTEGER<1-255>  Internal preference
!Distancia administrativa para rotas aprendidas via iBGP
 
[Router-bgp]preference 7 100 ?
INTEGER<1-255>  Local preference
!Distancia administrativa para rotas aprendidas via iBGP (locais)
 
[Router-bgp]preference 7 100 9 
```

### Configuração 

```
reset bgp x.x.x.x
executado em user-view
refresh bgp x.x.x.x import
força atualização de rotas recebidas
refresh bgp x.x.x.x export
força atualização de rotas anunciadas
```

### Troubleshooting de BGP

- Comandos para ver rotas recebidas e anunciadas
```
display bgp routing-table peer x.x.x.x received-routes
display bgp routing-table peer x.x.x.x advertised-routes
```

- MP-BGP / VPNv4
```
display bgp vpnv4 vpn-instance vpn-instance-name routing-table peer x.x.x.x received-routes
display bgp vpnv4 all routing-table peer x.x.x.x received-routes
```

- Rotas anunciadas
```
display bgp vpnv4 vpn-instance vpn-instance-name routing-table peer x.x.x.x advertised-routes
display bgp vpnv4 all routing-table peer x.x.x.x advertised-routes
```

- Fluxo prático de troubleshooting
  
 - Verificar vizinhança
   - confirmar se o peer BGP está estabelecido
- Conferir rotas recebidas
   - ver se o prefixo chegou no equipamento
- Conferir rotas anunciadas
  - validar se o prefixo está sendo exportado
- Conferir tabela de roteamento
  - checar se a rota BGP entrou na tabela IP
- Comparar com política/VRF
  - revisar filtros, route-policy, prefix-list, import/export e VPN-instance
- Aplicar refresh ou reset
  - quando necessário, para testar reaplicação das rotas
