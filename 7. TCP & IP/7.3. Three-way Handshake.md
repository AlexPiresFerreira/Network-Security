
Ao se iniciar uma conexão TCP é feito um handshake, que consiste no envio de 3 pacotes (SYN, SYN/ACK e ACK).

Em toda sessão TCP os envolvidos sorteiam um ISN (Initial Sequence Number) que deve ser incrementado, um para cada pacote, durante o handshake. Esse número apoia o processo de confirmação de recebimento de pacotes.



```mermaid
sequenceDiagram
    participant C as Cliente (Porta 51515)
    participant S as Servidor (Porta 80)

    Note over C,S: Início do Three-way Handshake
    C->>S: SYN (SEQ=100, ACK= - )
    S-->>C: SYN-ACK (SEQ=300, ACK=101)
    C->>S: ACK (SEQ=101, ACK=301)
    Note over C,S: Conexão TCP Estabelecida

    Note over C,S: Troca de Dados
    C->>S: Dados 1 (SEQ=101, ACK=301, Tamanho=200)
    S-->>C: Resposta (SEQ=301, ACK=301+200)
    C->>S: Dados 2 (SEQ=301, ACK=501, Tamanho=200)
    S-->>C: Resposta (SEQ=501, ACK=501+200)
    C->>S: Dados 3 (SEQ=501, ACK=701, Tamanho=200)
    S-->>C: Resposta (SEQ=701, ACK=701+200)

    Note over C,S: Finalização da Conexão (Four-way Handshake)
    C->>S: FIN (SEQ=701, ACK=901)
    S-->>C: FIN-ACK (SEQ=901, ACK=702)
    C->>S: ACK (SEQ=702, ACK=902)
    Note over C,S: Comunicação Finalizada
```
